# Document Analysis Pro

Use this skill when the user asks to analyze one or more documents deeply, not just short summarization.

## Workflow
1. Identify document paths and formats.
2. Extract text safely and verify extraction quality.
3. Build a document map:
- title
- sections
- key entities
- claims/evidence pairs
4. Produce outputs based on user goal:
- executive summary
- key findings
- open questions
- risks/contradictions
- action items
5. Always include source pointers (section/page/paragraph when possible).

## Output Template
- Goal
- Sources
- Summary
- Key Findings
- Risks and Gaps
- Recommended Next Steps
